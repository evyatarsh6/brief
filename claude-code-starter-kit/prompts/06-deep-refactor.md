# Prompt: Deep Refactor

```text
I want to refactor <AREA> without changing behavior.

Before editing:
1. Identify current behavior and existing tests.
2. Explain the refactor goal.
3. Propose small sequential refactor steps.
4. Identify risks and rollback strategy.
5. Do not mix feature changes with refactor.
6. Do not edit files until I approve the plan.

After implementation:
- Run tests/typecheck/lint.
- Summarize changed files.
- Confirm whether behavior changed.
```
