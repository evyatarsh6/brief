# Prompt: Create PRD

```text
Create docs/PRD.md based on the information we have.

Requirements:
- Separate facts from assumptions.
- Include background, problem statement, goals, non-goals, users, user flows, functional requirements, non-functional requirements, constraints, risks, open questions, and acceptance criteria.
- Use clear IDs for requirements, such as FR-001.
- Do not design the architecture yet.
- If information is missing, mark it as an open question instead of inventing it.
```
